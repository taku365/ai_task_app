音声入力機能 調査・実装報告

タスク管理アプリ（AI-task-app）/ feature/voice-input ブランチ
どの方法で作るか調べた
音声入力を実現する方法はいくつかあります。それぞれのメリット・デメリットを比べました。
方法 お金 精度 ブラウザ サーバー工事
Web Speech API 無料 ふつう Chrome・Edge のみ 不要
OpenAI Whisper API 従量課金 高い 全ブラウザ 必要
Google STT 従量課金 高い 全ブラウザ 必要
Azure Speech 従量課金 高い 全ブラウザ 必要
今回は Web Speech API を選びました。
理由は「お金がかからない・サーバーの改修が不要・すでにある AI 解析とそのままつながる」の3つです。 まずは低コストで試してみて、精度やブラウザ対応に問題が出てきたら Whisper API への切り替えを検討します。
そもそも「Web Speech API」って何？
むずかしい言葉が出てきますが、シンプルに言うとこういうものです。
🎙️
声をテキストに変えてくれる
マイクに向かって話すだけで、文字に変換してくれます。スマホの音声入力と同じしくみです。
🌐
ブラウザだけで動く
アプリをインストールしなくても、Chrome などのブラウザがそのまま対応しています。
💰
無料で使える
外部サービスへの課金なしで利用できます。何万回使っても費用はかかりません。
☁️
声はGoogleに送られる
Chrome の場合、音声データは Google のサーバーで処理されてテキストが返ってきます。
💡 たとえるなら
「Siri に話しかけたら文字にしてくれる」のと同じことを、アプリの中でできるようにしたものです。
今回のアプリでの使い方：マイクボタンを押して話し、停止ボタンを押すまでずっと録音し続けます。止めたタイミングで、まとめてテキストを AI に渡して解析します。
使ったときに何が起きているか
ボタンを押してからタスクが登録されるまでの流れです。
1
🎙️ マイクボタンをタップする
画面右下のフローティングマイクボタンをタップします。画面上部に「録音バー」が現れて、録音が始まります。
2
🗣️ 話す（Web Speech API が文字に変換）
「明日の午後3時までに田中さんに資料を送る」など、タスクの内容を話します。話した内容はリアルタイムで文字にどんどん変換・蓄積されています。
3
⏹️ 停止ボタンをタップする
録音バーが「解析中...」の表示に切り替わります。誤って別の操作をしないようにボトムナビも一時的に無効になります。
4
🤖 AI が内容を読み解く
文字起こしされたテキストを AI に渡します。AI が「タスク名・期限・担当者・優先度・かかる時間」を自動で読み取ります。
5
✅ タスクが自動で登録される
AI の解析結果をそのまま DB に保存。ボタン操作なしでタスク一覧にすぐ反映されます。録音バーも自動で消えます。
💡 たとえるなら
「Siri、明日の会議をリマインドして」と話しかけると登録されるのと同じ感覚で、タスクが登録できるイメージです。
どのファイルを変えたか
実際にコードを書き変えたファイルと、その内容です。
ファイル 何を変えたか
tasks.js ・音声入力専用の登録関数 voiceCreateTask() を新しく追加
・既存の textCreateTask() から AI 解析の処理を切り出して整理
・initVoiceInput() で録音バーの表示・非表示を制御
index.blade.php ・右下にフローティングマイクボタンを追加
・録音中バー（波形アニメーション・停止ボタン）を追加
app.css ・録音バーのデザイン（録音中・解析中の2パターン）を追加
・フローティングボタンのデザインを追加
TaskController.php ・AI に渡すプロンプトに「時間」の抽出ルールを追加
・「今日」「明日」「今週中」などの表現を実際の日付に変換してから渡すよう修正
これからの課題
使えるようにはなりましたが、まだ気になっている点があります。
  • Safari・Firefox では使えない Web Speech API は Chrome と Edge にしか対応していません。Safari や Firefox のユーザーが増えてきたら、Whisper API（全ブラウザ対応）への切り替えを検討します。
  • 聞き間違えたときの対処 音声認識が間違えると、そのままタスクに登録されてしまいます。「〇〇として登録しました」というお知らせを出して、すぐ編集できるよう誘導する仕組みを検討しています。
  • いろんな話し方での精度確認 専門用語が多い内容・早口・周囲がうるさい環境など、さまざまなケースで正しく認識できるか確認が必要です。
  • アプリを開かずに登録できるようにしたい 将来的には、ブラウザ拡張やブックマークレットとして音声ボタンを独立させ、アプリを開かなくても「話すだけでタスクを登録」できる体験を目指しています。
