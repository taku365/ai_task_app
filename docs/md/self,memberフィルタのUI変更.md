# UI仕様変更 & 時間選択機能 実装指示書

## 概要

`self` / `member` フィルタ画面のUIを、カード型からTodoist風のグループリスト型に変更する。
あわせて、タスクへの時間設定機能を実装する。

---

## 1. 時間選択機能の実装

### 1-1. 時間入力UI

タスク作成・編集画面に時刻ピッカーを追加する。

- 時（HH）と分（MM）をそれぞれ選択できる形式
- 時間は任意項目（設定しない場合は `null` で保存）
- 設定済みの時間はクリア（削除）できるようにする

### 1-2. データ保存

- 既存の日付フィールドとは別に、時間フィールドを持たせる
- 例：`due_date: "2026-03-12"`, `due_time: "11:30"` または `null`

---

## 2. フィルタ画面のUI変更（self / member）

### 2-1. レイアウト変更

| 項目 | 変更前 | 変更後 |
|------|--------|--------|
| レイアウト | カードUI（1枚ずつ独立したカード） | グループリストUI（日付ごとにグループ化） |
| 区切り | カード境界線 | 日付ヘッダー |

### 2-2. グループの種類と並び順

以下の順で上から表示する。

```
1. 期限切れ
2. 今日
3. 明日
4. 以降の日付（昇順）
```

### 2-3. 日付ヘッダーの表示形式

```
3月12日・今日・木曜日
3月13日・明日・金曜日
3月14日・土曜日
```

- 「今日」「明日」に該当する場合のみ、MM月DD日の後ろにラベルを付ける
- 曜日は常に表示する

### 2-4. 「期限切れ」グループ

- ヘッダーテキスト：`期限切れ`
- 並び順：期日が古い順（昇順）
- 日付フォーマット：`M月D日`（例：`3月6日`）
- 昨日の場合は `昨日` と表示（時間がある場合は `昨日 11:00`）

---

## 3. タスク表示ルール

### 3-1. 日付フォーマット

| 変更前 | 変更後 |
|--------|--------|
| `2026年3月12日` | `3月12日` |

### 3-2. 時間の表示

- 時間が設定されている場合のみ表示する
- 表示例：`11:30` / `15:00`
- 時間がない場合は表示しない（空欄・非表示）

### 3-3. グループ内の並び順

- 時間あり → 時間順（昇順）
- 時間なし → 時間ありの後ろに表示

---

## 4. 表示サンプル（参考）

```
期限切れ
  ○ タスクA
      3月6日
  ○ タスクB
      昨日 11:00

3月12日・今日・木曜日
  ○ タスクC（時間あり）
      11:30
  ○ タスクD（時間なし）

3月13日・明日・金曜日
  ○ タスクE
      15:00

3月14日・土曜日
  ○ タスクF
      13:00
```

---

## 5. 変更対象外

- `未割当` / `完了済み` フィルタのUIはそのまま
- ヘッダー・タブバー・FABボタンはそのまま
