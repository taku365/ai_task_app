## 1. 概要

各タブページ（自分 / メンバー / 未割当 / 完了 / すべて）に絞り込みフィルターを追加する。
検索専用ページは作成しない。

### 変更点まとめ

| 項目 | 変更前（初期案） | 変更後（採用案） |
|---|---|---|
| アクセス方法 | ボトムナビに「検索」タブを追加 | 各タブページ内にフィルターを設置 |
| フィルター表示 | 常時表示 | 🔍 アイコンタップで展開/折りたたみ |
| 対象タブ | 検索専用ページのみ | すべてのタブ（「すべて」タブを新規追加） |

---

## 2. タブ構成の変更

### 変更前

```
[自分] [メンバー] [未割当] [完了]
```

### 変更後

```
[自分] [メンバー] [未割当] [完了] [すべて]
```

「すべて」タブを末尾に追加する。「完了」タブは「完了」に短縮する。

---

## 3. 「すべて」タブの仕様

### 表示内容

- 担当者・完了状態に関わらず、**全タスクを表示**
- 未完了タスク・完了タスクが混在して表示される

### 表示順

- タスクの登録順（新しい順）

### 件数・ページネーション

- 1ページあたり **30件** を表示
- 30件を超える場合はページネーション（前へ / 次へ）で切り替える

### 完了タスクのUI

- 完了タスクは既存の完了UIをそのまま流用する
  - 緑のチェックアイコン
  - メタ情報（日時・担当者・優先度）はグレー表示
- 未完了タスクと完了タスクが視覚的に区別できる状態を維持する

---

## 4. 絞り込みフィルターの仕様

### 4-1. 表示/非表示

- 各タブページの上部に 🔍 アイコンボタンを設置
- タップするとフィルターパネルが展開、もう一度タップで折りたたむ
- フィルター適用中はボタンのテキストが「絞り込み中」に変わり、赤色で強調表示する

### 4-2. タブ別フィルター設定

| タブ | 担当者 | 優先度 | 期限 |
|---|---|---|---|
| すべて | すべて選択可 | 選択可 | 選択可 |
| 自分 | 自分（固定・変更不可） | 選択可 | 選択可 |
| メンバー | 自分以外のメンバーから選択 | 選択可 | 選択可 |
| 未割当 | すべて選択可 | 選択可 | 選択可 |
| 完了 | すべて選択可 | 選択可 | 選択可 |

### 4-3. 担当者フィルター

| 項目 | 内容 |
|---|---|
| UI | ドロップダウン（単一選択） |
| 「自分」タブ | ログインユーザー名で固定（変更不可） |
| 「メンバー」タブ | 自分以外のメンバー一覧から選択（デフォルト：すべて） |
| 「すべて」「未割当」「完了」タブ | メンバー全員から選択（デフォルト：すべて） |
| データソース | 既存の `MEMBERS` データを流用 |

> **単一選択とした理由:**
> 「特定の1人のタスクを探す」ユースケースが大半であり、複数選択は操作・実装ともに複雑度が増すため採用しない。

### 4-4. 優先度フィルター

| 項目 | 内容 |
|---|---|
| UI | ドロップダウン（単一選択） |
| 選択肢 | すべて（デフォルト）/ 高 / 中 / 低 / 指定なし |
| データソース | 既存の priority マスターを流用 |

### 4-5. 期限フィルター

| 項目 | 内容 |
|---|---|
| UI | ドロップダウン（単一選択） |
| 選択肢 | すべて（デフォルト）/ 今日 / 明日まで / 今週中 / 来週中 / 期限切れ / 期限なし |

> **【将来的な拡張案】**
> - **日付範囲指定：** 開始日〜終了日を自由に指定できる入力方式も検討可能。既存のカレンダー UI を流用することで実装できるが、操作ステップが増えるため今回は採用しない。
> - **長期プリセット：** 1ヶ月 / 2ヶ月 / 3ヶ月 / 半年 / 1年 などの選択肢追加も検討可能。ただし選択肢が増えすぎると迷いが生じるため、ニーズが出てきた段階で追加する。

---

## 5. フィルターの動作

- 複数条件を同時に指定した場合は AND 条件で絞り込む
- フィルター変更のたびにリアルタイムで結果を更新する
- 0件の場合は「該当するタスクが見つかりませんでした」を表示
- タスクをタップすると既存の詳細モーダルを開く

---

## 6. 実装スコープ

| 対象 | 作業内容 | 規模 |
|---|---|---|
| タブ | 「すべて」タブを先頭に追加 | 小 |
| 既存 Blade | 各タブページにフィルターパネルを追加 | 小〜中 |
| JS | フィルター展開/折りたたみ・絞り込みロジック・結果描画 | 中 |
| バックエンド | 既存APIに絞り込みパラメータを追加（LIKE検索 + 絞り込み） | 小 |
| DB マイグレーション | **不要**（既存カラムで対応可能） | - |

---

## 7. 既存機能との関係

| 機能 | 影響 |
|---|---|
| フィルタータブ（自分/メンバー/未割当/完了） | タブ自体は維持。「すべて」タブを先頭に追加 |
| タスク詳細モーダル | 変更なし。フィルター結果からもそのまま流用 |
| ボトムナビ | 変更なし（「検索」タブは追加しない） |
| カレンダー UI | 将来の期限範囲指定拡張時に流用予定 |
